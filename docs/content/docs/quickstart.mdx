## Prerequisites

- Docker and Docker Compose
- `make`
- `openssl` for local dev certificate files
- `jq` for command examples

## Steps

<Steps>
<Step>

### Create Local Dev Config

```bash
make dev-env
```

This creates `.env.dev` with local defaults and generates local root CA files
under `certs/` if they do not already exist.

</Step>
<Step>

### Start Atom

```bash
make up
```

This builds and starts:

- Atom REST and GraphQL at `http://localhost:18080`
- Atom Next UI at `http://localhost:3005`
- Postgres at `127.0.0.1:5432`

The default local admin password in `.env.dev` is `12345678`. Edit
`ADMIN_SECRET` in `.env.dev` before first boot if you want a different password.

</Step>
<Step>

### Watch Logs

```bash
make logs
```

Atom applies migrations automatically on startup.

</Step>
<Step>

### Log In

```bash
TOKEN=$(curl -s -X POST http://localhost:18080/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"identifier": "atom-admin", "secret": "12345678"}' \
  | jq -r .token)
```

</Step>
<Step>

### Create A Tenant, Device, And Channel

Use GraphQL at `POST http://localhost:18080/graphql`.

```bash
curl -s http://localhost:18080/graphql \
  -H "Authorization: Bearer $TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{
    "query": "mutation { createTenant(input: { name: \"factory-a\", route: \"factory-a\" }) { id name route } }"
  }'
```

Create an entity for a device:

```graphql
mutation {
  createEntity(input: {
    kind: device
    name: "meter-001"
  }) {
    id
    name
    kind
  }
}
```

Create a channel resource:

```graphql
mutation {
  createResource(input: {
    kind: "channel"
    name: "telemetry"
  }) {
    id
    kind
    name
  }
}
```

</Step>
<Step>

### Give Publish Access

Find the seeded `publish` action:

```graphql
{
  actions(objectKind: "resource", objectType: "resource:channel") {
    items {
      id
      name
    }
  }
}
```

Create a permission block for the channel:

```graphql
mutation {
  createPermissionBlock(input: {
    scopeMode: "object"
    objectId: "channel-resource-id"
    effect: allow
    actionIds: ["publish-action-id"]
  }) {
    id
    effect
    actions { name }
  }
}
```

Create a role and attach the permission block:

```graphql
mutation {
  createRole(input: {
    name: "channel-publisher"
    description: "Can publish telemetry"
  }) {
    id
    name
  }
}
```

```graphql
mutation {
  replaceRolePermissionBlocks(
    roleId: "role-id"
    permissionBlockIds: ["permission-block-id"]
  )
}
```

Assign the role to the device:

```graphql
mutation {
  createRoleAssignment(input: {
    subjectKind: entity
    subjectId: "device-entity-id"
    roleId: "role-id"
  }) {
    id
    subjectId
    roleId
  }
}
```

</Step>
<Step>

### Check Authorization

```graphql
mutation {
  authzCheck(input: {
    subjectId: "device-entity-id"
    action: "publish"
    resourceId: "channel-resource-id"
  }) {
    allowed
    reason
  }
}
```

Expected result:

```json
{
  "allowed": true,
  "reason": "allowed"
}
```

</Step>
</Steps>

## Makefile Commands

Run `make help` for the current target list.

| Command | What it does |
|---|---|
| `make dev-env` | Creates `.env.dev` with local defaults and local dev CA files. |
| `make up` | Builds and starts Postgres, Atom, and Atom UI. |
| `make logs` | Follows Atom backend and Atom UI logs. |
| `make down` | Stops the local Compose stack. |
| `make restart` | Stops, rebuilds, and starts the local Compose stack again. |
| `make build` | Builds Atom backend and Atom UI images. |
| `make atom-build` | Builds only the Atom backend image. |
| `make ui-build` | Builds only the Atom UI image. |
| `make docker-build` | Builds the raw Atom Docker image using Makefile variables. |
| `make docker-build-release` | Builds the raw release Docker image. |
| `make docker-build-dev` | Builds the raw dev Docker image. |

Common overrides:

```bash
# Use another env file
DEV_ENV_FILE=.env.local make up

# Use different host ports
POSTGRES_HOST_PORT=55432 ATOM_HTTP_PORT=28080 ATOM_UI_HTTP_PORT=3006 make up

# Start only backend services, without the UI profile
COMPOSE_PROFILES="--profile default" make up
```

## Direct Cargo Development

Use this path when you want to run the Rust service directly on your host
instead of inside the Compose backend:

```bash
cp .env.example .env
# set ADMIN_SECRET on first boot
# set ATOM_CERTS_ENABLED=false for a minimal local run, or configure cert paths

docker compose --env-file .env up -d postgres
cargo run
```

The direct Cargo backend listens on `http://localhost:8080` by default. The
Makefile Compose backend listens on `http://localhost:18080`.

## What Just Happened

1. You created an entity for a device.
2. You created a resource for a channel.
3. You created a permission block that allows `publish` on that channel.
4. You put the permission block inside a role.
5. You assigned the role to the device.
6. Atom returned allow when asked whether the device can publish.

See [Atom In Simple Words](/simple-words) for the layman explanation, and [Access Control](/access-control) for the full model.
