## GET /entities/:id/access

Returns all resources an entity can access, showing which roles, capabilities, and policy bindings grant the access â€” including access inherited through group memberships.

## Query parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `tenant_id` | UUID | â€” | Filter resources by tenant |
| `resource_kind` | string | â€” | Filter resources by kind (e.g. `channel`, `device`) |
| `action` | string | â€” | Filter by specific action/capability name |
| `effect` | `allow` \| `deny` | â€” | Filter by effect |
| `limit` | int | 20 | Results per page (1-100) |
| `offset` | int | 0 | Pagination offset |

## Response fields

| Field | Type | Description |
|---|---|---|
| `resource` | object | Resource details (id, kind, name, tenant_id) |
| `effect` | `allow` \| `deny` | Whether this is an allow or deny |
| `scope_kind` | string | How broad the scope is |
| `policy_id` | UUID | The policy binding that creates this access |
| `grant.kind` | `capability` \| `role` | Grant type |
| `grant.role` | object \| null | Role info if applicable |
| `grant.capabilities` | array | Capabilities granted (expanded from role if applicable) |
| `conditions` | object | ABAC conditions on the binding |
| `via` | string | `"direct"` or `"group:<group-name>"` |

## Scope expansion

- **`scope_kind = resource`** â€” one item per directly referenced resource.
- **`scope_kind = resource_kind`** â€” one item per resource matching that kind.
- **`scope_kind = all`** â€” one item per resource (within tenant filter if provided).

Pagination applies to the expanded resource list.

## Use cases

| Query | Endpoint |
|---|---|
| What can Alice access in tenant 1? | `GET /entities/aaa/access?tenant_id=t1` |
| What can Alice access across all tenants? | `GET /entities/aaa/access` |
| What channels can Alice access in tenant 1? | `GET /entities/aaa/access?tenant_id=t1&resource_kind=channel` |
| What can Alice read? | `GET /entities/aaa/access?action=read` |
| What is Alice denied? | `GET /entities/aaa/access?effect=deny` |
