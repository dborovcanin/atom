## GET /resources/:id/access

The inverse of entity access тАФ returns all entities that have access to a specific resource, showing how each one gets that access.

## Query parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `action` | string | тАФ | Filter by action/capability name |
| `entity_kind` | string | тАФ | Filter by entity kind (e.g. `human`, `device`, `service`) |
| `effect` | `allow` \| `deny` | тАФ | Filter by effect |
| `limit` | int | 20 | Results per page (1-100) |
| `offset` | int | 0 | Pagination offset |

## How policies are matched

A policy binding covers this resource if:

1. `scope_kind = all` тАФ covers every resource.
2. `scope_kind = resource_kind` AND `scope_ref` equals this resource's `kind`.
3. `scope_kind = resource` AND `scope_ref` equals this resource's `id`.

For bindings with `subject_kind = group`, group membership is expanded тАФ each entity in the group appears as a separate item with `via: "group:<name>"`.

## Use cases

| Query | Endpoint |
|---|---|
| Who can access this resource? | `GET /resources/r1/access` |
| Which humans can write to this resource? | `GET /resources/r1/access?action=write&entity_kind=human` |
| What deny policies apply to this resource? | `GET /resources/r1/access?effect=deny` |
| Which devices can publish to this channel? | `GET /resources/r1/access?action=publish&entity_kind=device` |
